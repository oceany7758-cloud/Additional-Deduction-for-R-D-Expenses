# 产品需求文档：项目信息管理

> 模块归属：研发费用加计扣除管理系统
> 版本：V1.0
> 日期：2026-06-12
> 范围：一期

---

## 一、模块概述

项目信息管理是整个研发费用加计扣除系统的基础数据中枢。它管理研发项目全生命周期的基本信息，并维护项目关联的人员、资产、组织、档案四类主数据。

**核心定位**：所有辅助账编制、费用归集分摊、税务申报输出，都以项目为最小核算单元。项目信息的准确性直接决定后续所有计算的正确性。

**包含五个子模块**：
1. 研发项目信息 — 项目全生命周期管理（核心）
2. 研发人员信息 — 研发人员主数据维护
3. 研发资产信息 — 研发使用的仪器设备等主数据维护
4. 研发组织信息 — 合作方/受托方组织信息维护
5. 研发档案信息 — 研发备查资料档案管理

**数据维护方式**：五个子模块均支持接口对接、手工录入、批量导入三种方式。接口对接场景下，部分字段可配置是否允许二次手工修改。

## 二、研发项目信息

### 2.1 页面结构

项目信息管理包含两个页面：
- **列表页**：项目概览，支持筛选、搜索、批量操作
- **详情编辑页**：单个项目的完整信息编辑，包含基本信息、费用时间、委托研发、项目来源、关联信息五个区块

### 2.2 列表页

#### 用户场景

研发管理人员需要快速找到目标项目，查看项目关键状态，并执行新增、编辑、删除、资本化结转等操作。

#### 查询条件

系统提供11个查询条件。其中项目名称、项目状态、支出类型、项目期间四个条件为**默认筛选条件**（页面加载时即作为查询条件生效），其余条件默认不启用，用户可手动展开使用。

查询支持模糊搜索（项目名称、项目编号），下拉条件支持单选和多选。

#### 列表展示

列表共展示13个字段，核心信息一目了然：

- **项目基本标识**：项目编号、项目名称
- **项目分类**：技术领域（对应《国家重点支持的高新技术领域》三级目录）、项目来源
- **状态信息**：项目状态（已立项 / 研发中 / 已结项）、辅助账状态（同步自账簿管理：待建账 / 已启用 / 已关账）
- **费用信息**：支出类型（费用化 / 资本化 / 混合型）、组织形式、项目总预算（万元）、项目实际支出（万元，自动从辅助账取数 = 费用化支出 + 资本化支出）
- **时间信息**：项目开始日期、项目结束日期
- **完成情况**：未完成 / 已完成-研发成功 / 已完成-研发失败

#### 操作

| 操作 | 说明 |
|------|------|
| 新增 | 进入项目详情编辑页，创建新项目 |
| 修改 | 选中项目后进入详情编辑页，编辑已有信息 |
| 删除 | 选中项目后确认删除 |
| 资本化结转 | 选中满足前置条件的项目，发起资本化结转无形资产流程。前置条件：项目状态为"已结项"，辅助账状态为"已关账"，资本化阶段为"归集核算" |
| 导入 | 批量导入项目数据（Excel） |
| 导出 | 将当前查询结果导出为文件 |

### 2.3 详情编辑页

详情页包含30个字段，按业务逻辑分为五个区块。必填字段共11个（含6个条件必填），另有多字段由系统自动赋值或从关联数据自动取数。

#### 区块一：基本信息（9个字段）

用于描述项目的核心身份和研究内容。

**必填字段**：项目编号（需包含字母）、项目名称、项目状态、组织形式、主要研究内容、创新点

**下拉枚举**：
- 项目状态：已立项 → 研发中 → 已结项（状态流转）
- 组织形式：自主研发 / 委托研发 / 合作研发 / 集中研发
- 技术领域：对应《国家重点支持的高新技术领域》三级目录

#### 区块二：费用与时间（9个字段）

用于记录项目的预算、实际支出和关键时间节点。

**必填字段**：支出类型、项目总预算

**条件必填**：
- 项目开始日期：当项目状态为"研发中"或"已结项"时必填
- 项目结束日期：当项目状态为"已结项"时必填
- 资本化开始日：当支出类型为"混合型"时必填
- 完成情况：当项目状态为"已结项"时必填（枚举：未完成 / 已完成-研发成功 / 已完成-研发失败）

**系统自动字段**：
- 项目实际支出：从研发支出辅助账自动取数，= 费用化支出 + 资本化支出。金额为项目全周期、全口径合计
- 资本化阶段：系统自动赋值，详见下方"资本化阶段自动赋值逻辑"

#### 区块三：委托研发（2个字段，条件显示）

当且仅当组织形式为"委托研发"时，显示此区块，两个字段均为必填。

- 委托研发的关联关系：关联方 / 非关联方
- 是否委托境外：委托境内 / 委托境外

**联动规则**：当从研发组织信息中关联选择合作组织时，这两个字段根据所选组织的属性自动赋值，无需手工填写。

#### 区块四：项目来源（5个字段）

用于记录项目的来源渠道和管理归属。

**必填字段**：项目来源、管理用途（可多选：加计扣除 / 高新认定 / IPO上市 / 其他用途）

**条件必填链**：
- 项目级别：当项目来源为"政府科技项目"时必填（市级 / 省级 / 国家级）
- 科技计划项目编号：当项目级别为"省级"或"国家级"时必填

**枚举**：项目活动类型（基础研究 / 应用研究 / 试验发展 / 成果应用 / 科技服务）

#### 区块五：关联信息（5个字段，含4个浮动子表）

用于管理项目与其他主数据的关联关系。

- **辅助账状态**（只读）：自动同步账簿管理中对应账簿的状态
- **项目参与人员**（必填）：通过关联选择从研发人员信息中选取，需额外设置研发分工、参与项目起止日期。工时已计入项目的人员不允许解除关联
- **项目使用资产**（必填）：通过关联选择从研发资产信息中选取。折旧、摊销费已计入项目的资产不允许解除关联
- **项目参与组织**（合作研发/委托研发/集中研发时必填）：通过关联选择从研发组织信息中选取
- **项目备查资料**（自动归集）：从研发档案信息中自动归集本项目相关的档案，不可手工编辑

### 2.4 资本化结转

资本化结转是将研发项目资本化阶段的支出结转为无形资产的操作。

**触发条件**：项目状态="已结项"，辅助账状态="已关账"，资本化阶段="归集核算"。

**操作流程**（8步自动化）：
获取费用化阶段辅助账数据 → 获取资本化阶段辅助账数据 → 计算资本化金额 → 生成结转凭证 → 更新资本化阶段为"完成结转" → 生成无形资产卡片 → 关联无形资产摊销计算 → 更新辅助账状态

### 2.5 资本化阶段自动赋值逻辑

系统根据项目属性和操作进度自动维护资本化阶段状态，无需手工干预：

- **支出类型 = 费用化**：资本化阶段始终为"不适用"
- **支出类型 = 资本化 或 混合型**：
  - 资本化开始日之前 → "未开始"
  - 资本化开始日起，至资本化结转操作完成前 → "归集核算"
  - 资本化结转操作完成后 → "完成结转"

### 2.6 科目映射

项目信息中涉及三种科目类型，分别对应不同的会计核算体系：

| 科目类型 | 对应科目 | 用途 |
|----------|---------|------|
| 成本科目 | 研发费用 | 用于成本归集 |
| 损益科目 | 管理费用-研发费 | 用于损益核算 |
| 资产科目 | 无形资产 | 用于资本化项目结转 |

### 2.7 研发分工

项目参与人员需设置研发分工，枚举值为：研究、技术、辅助、管理、服务、其他。

## 三、研发人员信息

### 3.1 页面结构

单列表页，管理所有参与研发活动的人员信息。

### 3.2 查询条件

提供4个查询条件，均为默认筛选条件（页面加载时即生效）：工号、姓名、用工类别、任职期间。

### 3.3 列表展示

共15个字段：

- **人员身份**（必填）：工号、姓名、性别、身份证号、学历、专业
- **职称信息**（必填）：职称名称（38项字典）、职称类别（根据职称名称自动匹配，无需手工填写）
- **职务信息**（必填）：职务、用工类别
- **其他信息**：公司名称、任职期起、任职期止（同一人可记录多段任职期间，默认显示最近一段）、研发投入工时（二期，显示最近3年工时数，点击展开年度明细和相关项目信息）

### 3.4 联动规则

职称名称与职称类别存在自动联动关系。用户选择职称名称后，系统根据预设的38项映射表自动填充职称类别。映射规则覆盖教授、工程师、农艺师、兽医师、畜牧师、实验师、技师等多个系列，对应高级职称（正高）、高级职称（副高）、中级职称、初级职称（助理级）、初级职称（员级）、高级技工、中级技工、初级技工、其他等类别。

### 3.5 操作

支持新增、修改、删除、导入、导出五个操作。

**业务约束**：工时已计入项目的人员，不允许解除其与项目的关联关系。

### 3.6 数据维护

接口对接方式下，职称名称和用工类别两个字段可配置是否支持二次手工修改。

## 四、研发资产信息

### 4.1 页面结构

单列表页，管理研发活动中使用的各类资产（固定资产和无形资产）。

### 4.2 查询条件

提供8个查询条件：资产编号、资产名称、资产类别一级（固定资产/无形资产）、资产类别二级（按一级联动）、取得方式、使用状态、用途分类、税会折旧摊销差异（是/否）。

### 4.3 列表展示

共19个字段：

- **资产标识**（必填）：资产编号、资产名称
- **资产分类**：规格型号、资产类别一级、资产类别二级（联动一级：固定资产→设备/仪器；无形资产→软件著作权/专利）、取得方式
- **关联项目**：研发项目编码、公司名称、使用部门、使用状态
- **折旧信息**：会计原值、会计残值率、会计折旧年限、会计折旧方法、开始使用日期
- **税务差异**（必填）：税会折旧摊销差异（是/否）
- **用途信息**（必填）：用途分类
- **关联项目**（二期）：所涉研发项目数量，点击展开关联项目的基本信息和分摊费用金额
- **备注**

### 4.4 资产类别联动

资产类别一级和二级之间存在联动关系：
- 固定资产 → 设备、仪器
- 无形资产 → 软件著作权、专利

### 4.5 操作

支持新增、修改、删除、导入、导出五个操作。

**业务约束**：折旧、摊销费已计入项目的资产，不允许解除其与项目的关联关系。

### 4.6 数据维护

接口对接方式下，税会折旧摊销差异和用途分类两个字段允许手工修改。

## 五、研发组织信息

### 5.1 页面结构

单列表页，记录与研发活动相关的合作方、受托方或集团成员单位的组织信息。

### 5.2 查询条件

提供5个查询条件：组织编号、组织名称、组织类型（研究机构/高等学校/企业单位/个人）、国别类型（境内/境外）、关联关系（关联方/非关联方）。

### 5.3 列表展示

共8个字段：

- **组织标识**（必填）：组织编号、组织名称、证件号码
- **组织属性**（必填）：组织类型、国别类型、关联关系
- **关联信息**（二期）：研发参与项目数量
- **备注**

### 5.4 操作

支持新增、修改、删除、导入四个操作（不含导出）。

### 5.5 与其他模块的联动

组织信息被研发项目信息中的"项目参与组织"子表关联引用。当项目选择委托研发的组织后，项目详情中的"委托研发的关联关系"和"是否委托境外"两个字段根据所选组织的属性自动赋值。

## 六、研发档案信息

### 6.1 页面结构

单列表页，管理系统外部产生的各类研发相关备查资料档案。

### 6.2 查询条件

提供9个查询条件。其中档案类别小类、项目编号、档案日期、档案用途为**基本筛选条件**（业务上最常用的筛选维度），其余条件按需使用。

档案类别大类和小类之间存在联动关系，选择大类后小类列表自动过滤。

### 6.3 列表展示

共19个字段：

- **档案标识**（必填）：档案编号（命名规则可配置分段、位数、字符类型）、档案类别大类、档案类别小类（24种子类型，与大类联动）、档案名称
- **档案关联**：源文件编号、档案日期、项目编号（关联项目，允许多选，展示"项目编号+项目名称"，分号分割）、来源部门、附件（下载/预览）
- **档案用途**（必填）：可多选（加计扣除/高企认定/IPO/其他）。每个档案类别小类与档案用途建立关联关系，新增档案时根据小类自动填充默认用途，可手工修改
- **管理信息**：档案介质（纸质/电子/混合）、存放地点、保密等级（普通/内部/机密）、归档人、归档日期
- **修改追踪**：修改人、修改日期、修改原因
- **备注**

### 6.4 操作

支持新增、修改、删除、批量导出、上传附件五个操作。上传附件时需按档案类别分类上传。

### 6.5 档案类别与用途联动

档案类别小类共24项，分布在5个大类下：

- **项目立项**（4项）：立项决议、项目计划书、可行性研究报告、项目组编制及人员名单
- **项目过程**（8项）：委托研发合同、合作研发合同、费用分摊协议、工时记录台账、设备使用记录、研发领料单、项目进展情况报告、委托研发付款凭证
- **项目核算**（7项）：资本化批准文件、费用分摊明细表、辅助账调整说明、研发支出辅助账、辅助账汇总表、研发费用明细申报表、研发支出明细情况说明
- **项目结题**（3项）：结题报告、项目经费决算报告、成果证明
- **政府监管**（3项）：鉴定证明、资质证明、税务事项通知书

每个小类与档案用途建立默认关联，新增档案时系统根据所选小类自动填充默认的档案用途，用户可在此基础上手工多选补充。

### 6.6 与其他模块的联动

档案信息被研发项目信息中的"项目备查资料"子表自动归集引用，展示本项目关联的所有档案资料。

## 七、数据关联关系

五个子模块之间的数据关系如下：

**研发项目信息**作为主表，与其余四个子模块建立关联：

| 关联子表 | 关系 | 关联方式 | 业务约束 |
|----------|------|---------|---------|
| 项目参与人员 | 1:N | 从研发人员信息关联选择 | 工时已计入项目的人员不可解除关联 |
| 项目使用资产 | 1:N | 从研发资产信息关联选择 | 折旧/摊销费已计入项目的资产不可解除关联 |
| 项目参与组织 | 1:N | 从研发组织信息关联选择 | 选择后委托研发字段自动赋值 |
| 项目备查资料 | 1:N | 从研发档案信息自动归集 | 不可手工编辑 |

**反向关联**：
- 研发资产信息 → 研发项目信息：N:M（所涉研发项目，二期实现）
- 研发档案信息 → 研发项目信息：N:M（项目编号，允许多选）

**项目与账簿管理**：1:1关系，辅助账状态同步自账簿管理。

| 项目来源 = "政府科技项目" | 项目级别 | 必填 |
| 项目级别 = "省级"或"国家级" | 科技计划项目编号 | 必填 |

## 九、系统自动赋值字段汇总

| 字段 | 赋值逻辑 | 数据来源 |
